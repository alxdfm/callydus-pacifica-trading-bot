# Sprint 2: Designer Tasks

## Sprint Objective
Deliver the complete experience of choosing, comparing, reviewing, and activating presets, ready for development handoff.

## Scope
- final cards for the 3 presets
- short comparison between presets
- selected preset review panel
- activation block
- desktop and mobile states of the presets screen

## Final Sprint Deliverables
- complete presets desktop screen
- complete presets mobile screen
- final `Safer`, `Balanced`, and `More active` cards
- summary comparator
- review panel with editable fields
- final activation CTA with states
- Sprint 2 minimum handoff

## Task D2.1: Define final presets screen structure

### Objective
Close the macro structure of the Presets screen for desktop and mobile.

### Scope
- header
- presets grid
- comparator
- preset review
- activation block

### Activities
- organize the visual hierarchy of the page
- define block order on desktop
- define block order on mobile
- validate the relationship between comparison, review, and activation
- confirm the screen does not feel like a technical form

### Deliverables
- final visual architecture of the presets screen

### Dependencies
- Sprint 1 outputs

### Done criteria
- the structure clearly supports choice, review, and activation
- block order is clear on desktop and mobile
