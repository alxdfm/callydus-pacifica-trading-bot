# Decisão: Manter signing.ts isolado — não alterar sem decisão

**Data:** 2025-05-04  
**Status:** aceita  
**Autor:** Callydus

## Contexto

O protocolo de assinatura da Pacifica é específico: campos do JSON ordenados recursivamente em ordem alfabética, serializados compactamente, assinados com Ed25519, resultado em Base58. Qualquer desvio gera rejeição silenciosa da API.

## Opções consideradas

| Opção | Prós | Contras |
|-------|------|---------|
| Reescrever em TS idiomático | estilo uniforme | risco alto de bug de ordering |
| Manter como está, mover de pasta | zero risco de regressão | estilo não-uniforme — aceito |

## Decisão tomada

**Manter `signing.ts` como está, mover apenas para `exchange/pacifica/signing.ts`.**

## Consequências

- Assinatura continua funcionando sem risco de regressão
- Arquivo não segue convenções de estilo — aceito como exceção documentada
- Revisitar se Pacifica anunciar mudança no protocolo via changelog
